## 핵심 발견사항 TOP 5

### 1. 메뉴 구조 재편성 필요 (관리자)
- heading 그룹 없이 10개 메뉴 나열 → 가독성 저하
- 통계가 예약보다 위에 위치 → 우선순위 오정렬
- **제안**: 핵심운영 / 분석·피드백 / 매장설정 / 구독·결제 4개 그룹

### 2. 버그 수준 UX 문제 3건 (고객)
- 리뷰 작성 시 예약번호 수동 입력 강제
- 비회원 예약 전화번호 포맷팅 누락
- 프로필 수정 시 새 탭 열림 → 예약 데이터 유실 위험

### 3. 레이아웃 일관성 부재 (고객)
- 7개 booking 페이지의 max-width 설정 방식이 전부 다름
- 일부 페이지는 full-width로 레이아웃 깨짐
- **제안**: booking 레이아웃 레벨에서 통일

### 4. Materio 컴포넌트 저활용 (전체)
- `VStepper`, `VDatePicker`, `VCarousel` 등 미사용 → 커스텀 코드 ~270줄 절감 가능
- `VTimeline`, `VBadge`, `VBottomNavigation` 등 UX 향상 컴포넌트 미활용

### 5. 디자인 일관성 부족 (전체)
- 삭제 다이얼로그 3곳 중복, 스타일 불일치
- 빈 상태 UI 페이지마다 다름
- 통계 카드 구현 방식 불일치

```

Week 4+: Phase D (모바일·고급)

```
