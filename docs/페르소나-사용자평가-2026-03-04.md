## 2. 태스크 시나리오별 평가

### 2.1 회원가입/로그인

#### Persona A (김민지 - 소규모 샵 사장님) | 점수: 6.0/10

**플로우**: 랜딩페이지 → "무료로 시작하기" → 회원가입(/register) → 초기설정(/shop-admin/setup) → 대시보드

**Pain Points**:
- 회원가입 페이지 상단에 플랜 선택(FREE/PAID)이 있어 서비스를 경험하기 전에 선택을 강요받음 → 불안감 유발
- 비밀번호 강도 표시기 없음 (최소 8자 규칙만 존재)
- 소셜 로그인은 로그인 페이지에만 있고 회원가입 페이지에는 없음 → 불일치
- 초기설정에서 "나중에 설정" 기능은 좋으나, "30일 무료 체험" 안내가 회원가입이 아닌 설정 단계에서만 표시됨

**개선 제안**:
1. 플랜 선택을 온보딩 이후로 이동, 기본값은 트라이얼/무료
2. 회원가입 페이지에도 소셜 로그인 옵션 추가
3. 비밀번호 강도 표시기 추가

#### Persona B (이소연 - 프리랜서) | 점수: 6.5/10

- 모바일에서 2열 레이아웃이 1열로 올바르게 전환됨
- 약관 동의 체크박스와 인라인 링크(LegalDialog.vue)는 잘 작동
- 소셜 계정으로 직접 가입 불가 → 이메일/비밀번호 가입 후 소셜 연동

#### Persona C (박지훈 - 고객) | 점수: 8.5/10

- 고객 로그인(/booking/login)이 소셜 전용 (카카오/네이버/구글) → 편리함
- 신규 사용자는 프로필 페이지로 리다이렉트되어 전화번호 등록 → 잘 설계됨
- 비회원 예약 옵션("비회원으로 예약" 버튼) → 첫 사용자 마찰 최소화

#### Persona D (최은지 - 대형 살롱) | 점수: 5.5/10

- 회원가입 시 플랜별 직원 수 제한이 표시되지 않음 (FREE는 1명만 가능)
- 가격 비교 페이지(/pricing)가 회원가입 플로우에서 분리되어 있음

### 2.2 서비스 관리

#### Persona A (김민지) | 점수: 7.0/10

- 카드 그리드 레이아웃으로 서비스 표시 (`cols="12" sm="6" md="4" lg="3"`)
- 카테고리 관리가 별도 다이얼로그 → 소규모 샵에는 불필요한 복잡도
- 서비스 등록 폼에 필드가 많으나 단계별 위저드 없음
- 활성/비활성 토글(VSwitch)은 직관적

#### Persona B (이소연) | 점수: 6.0/10

**핵심 문제**: 서비스 이미지 업로드 기능 없음 → 네일아티스트에게 치명적
- 모바일에서 카드 레이아웃은 잘 작동
- 중첩 다이얼로그(서비스 폼 → 카테고리 관리)가 작은 화면에서 답답할 수 있음

#### Persona D (최은지) | 점수: 6.5/10

- 구독 기반 서비스 수 제한이 UI에 잔여 용량으로 표시되지 않음
- 대량 작업(복수 서비스 활성/비활성) 기능 없음
- 유사 서비스 복제 기능 없음

### 2.3 예약 관리

#### Persona A (김민지) | 점수: 7.5/10

**리스트 뷰**:
- 데스크톱은 VDataTable, 모바일은 MobileListView → 좋은 반응형 전략
- 상태 변경이 VMenu로 되어 있으나 옵션이 너무 많음 (PENDING, CONFIRMED, COMPLETED, CANCELLED, NO_SHOW)
- 대량 상태 변경 기능은 강력하나 비기술 사용자에게 혼란 가능

**캘린더 뷰**:
- FullCalendar + 사이드바(날짜 선택기, 상태 필터, 직원 필터) → 기능 풍부
- 1인 샵에서 직원 필터는 불필요한 노이즈

#### Persona B (이소연) | 점수: 7.0/10

- 캘린더 뷰가 프리랜서에게 자연스러운 선택
- 대시보드의 미할당 예약 알림은 유용
- 실시간 알림/푸시 알림 메커니즘 부재 → 능동적으로 앱을 확인해야 함

#### Persona C (박지훈 - 고객) | 점수: 8.0/10

- 멀티스텝 위저드: 서비스 선택 → 직원 선택 → 날짜 → 시간 → 고객정보 → 확인
- 세션 스토리지 백업으로 플로우 상태 보존 → 좋은 설계
- 재예약(Rebook) 기능으로 쉬운 재예약 가능
- 대기 시간/순서 정보 없음

#### Persona D (최은지) | 점수: 7.0/10

- 직원별 색상 코딩 캘린더는 4명 직원 관리에 우수
- 개별 직원 스케줄 뷰 없음 → 4명 이상이면 시각적으로 복잡
- 드래그앤드롭 예약 재조정 불가

## 4. 핵심 인사이트

### 강점
1. **일관된 반응형 디자인**: 거의 모든 페이지가 `useDisplay()`와 `MobileListView`로 적절한 모바일/데스크톱 전환 구현
2. **체계적 상태 관리**: Pinia 스토어가 깔끔하게 분리 (auth, customer-auth, booking, subscription 등)
3. **이중 인증 시스템**: 관리자/고객 인증이 서로 다른 토큰으로 깔끔하게 분리
4. **구독 인식 UI**: subscriptionStore 기반 기능 제한이 일관되게 적용
5. **에러 코드 매핑**: 백엔드 에러 코드(CP001-004, SV004-005 등)가 한국어 메시지로 적절히 매핑
6. **빈 상태 처리**: 대부분의 페이지에 적절한 빈 상태 표시 (아이콘, 설명, 액션 버튼)
7. **모바일 하단 네비게이션**: BookingBottomNav으로 모바일 네이티브 느낌의 네비게이션

### 약점
1. **대형 파일 크기**: reserve.vue(67KB+), 매장 상세(52KB+) → 컴포넌트 분리 필요
2. **실시간 업데이트 없음**: WebSocket/SSE 미적용 → 관리자가 수동 새로고침 필요
3. **서비스 이미지 미지원**: 뷰티/네일 업종에 치명적
4. **FREE 플랜 통계 블러**: 사용자를 유도하기보다 좌절감 유발 가능
5. **고객 커뮤니케이션 채널 없음**: 앱 내 메시지 기능 부재
6. **캘린더 드래그앤드롭 없음**: 읽기 전용 캘린더
