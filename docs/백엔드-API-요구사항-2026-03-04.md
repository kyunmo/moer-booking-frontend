**작성일**: 2026-03-04
**작성 근거**: Phase 1~6 개선 작업 중 프론트엔드에서 필요하나 백엔드에 미구현된 기능 목록

## 개요

### 기존 API 기반 구조

| 항목 | 내용 |
|------|------|
| Base URL | `http://localhost:8080/api` |
| 인증 방식 | Bearer JWT (`Authorization: Bearer {token}`) |
| 관리자 공통 헤더 | `X-Business-Id: {businessId}` (슈퍼관리자 전용) |
| 응답 포맷 | `{ success: boolean, data: T, error: { code, message } }` |
| 에러 코드 체계 | 도메인 접두어 + 3자리 숫자 (예: A001, SV001, TR001) |

### 기존 에러 코드 도메인 체계

| 접두어 | 도메인 |
|--------|--------|
| A | 인증(Auth) |
| U | 사용자(User) |
| B | 매장(Business) |
| SV | 서비스(Service) |
| ST | 직원(Staff) |
| R | 예약(Reservation) |
| RV | 리뷰(Review) |
| C | 고객(Customer) |
| CP | 고객 프로필/예약(Customer Profile) |
| TR | 체험판/구독(Trial) |
| SL | 구독 제한(Subscription Limit) |
| IQ | 조회 제한(Inquiry/Rate Limit) |
| IMG | 이미지(Image) - 신규 제안 |
| NTF | 알림(Notification) - 신규 제안 |
| CRM | 고객 CRM - 신규 제안 |
| STAT | 통계(Statistics) - 신규 제안 |
| PUB | 공개(Public) - 신규 제안 |

### 1. 서비스 이미지 업로드 API

#### 1-1. 서비스 이미지 추가

**기능명**: 서비스 이미지 업로드

**기능 설명**: 서비스에 대표 이미지를 최대 3장까지 업로드합니다. 네일아티스트, 헤어 디자이너 등 포트폴리오 성격의 이미지를 서비스에 직접 연결하여, 고객이 예약 전 서비스 결과물을 미리 확인할 수 있습니다.

**입력값**

- Method: `POST`
- Path: `/api/businesses/{businessId}/services/{serviceId}/images`
- Path Variables:
  - `businessId` (Long, 필수): 매장 ID
  - `serviceId` (Long, 필수): 서비스 ID
- Headers:
  - `Authorization: Bearer {accessToken}`
  - `Content-Type: multipart/form-data`
- Request Body (multipart/form-data):
  - `file` (MultipartFile, 필수): 이미지 파일 (jpg, jpeg, png, webp)
  - `sortOrder` (Integer, 선택): 표시 순서 (기본값: 현재 이미지 수 + 1)
  - `caption` (String, 선택): 이미지 설명 (최대 100자)

**출력값**

```json
{
  "success": true,
  "data": {
    "id": 101,
    "serviceId": 42,
    "imageUrl": "https://cdn.moer.kr/services/42/images/abc123.webp",
    "thumbnailUrl": "https://cdn.moer.kr/services/42/images/abc123_thumb.webp",
    "sortOrder": 1,
    "caption": "젤 네일 시술 결과물",
    "createdAt": "2026-03-04T10:00:00"
  }
}
```

**예외 처리**

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 400 | IMG001 | 지원하지 않는 파일 형식입니다. (jpg, jpeg, png, webp만 허용) | 파일 형식 오류 |
| 400 | IMG002 | 파일 크기가 초과되었습니다. (최대 10MB) | 파일 용량 초과 |
| 400 | IMG003 | 서비스 이미지는 최대 3장까지 등록 가능합니다. | 이미지 최대 개수 초과 |
| 403 | B001 | 해당 매장에 대한 권한이 없습니다. | 권한 없음 |
| 404 | SV001 | 서비스를 찾을 수 없습니다. | serviceId 없음 |

**기타 참고사항**
- 업로드 후 자동으로 webp 변환 및 썸네일(320x320) 생성 권장
- 프론트엔드 호출 위치: `src/pages/shop-admin/services/components/ServiceFormDialog.vue`

#### 1-2. 서비스 이미지 목록 조회

**기능명**: 서비스 이미지 목록 조회

**기능 설명**: 특정 서비스에 등록된 이미지 목록을 조회합니다. 관리자 화면(서비스 편집) 및 고객 예약 화면(서비스 상세) 양쪽에서 사용합니다.

**입력값**

- Method: `GET`
- Path: `/api/businesses/{businessId}/services/{serviceId}/images`
- Path Variables:
  - `businessId` (Long, 필수)
  - `serviceId` (Long, 필수)
- Headers: `Authorization: Bearer {accessToken}`

**출력값**

```json
{
  "success": true,
  "data": [
    {
      "id": 101,
      "serviceId": 42,
      "imageUrl": "https://cdn.moer.kr/services/42/images/abc123.webp",
      "thumbnailUrl": "https://cdn.moer.kr/services/42/images/abc123_thumb.webp",
      "sortOrder": 1,
      "caption": "젤 네일 시술 결과물",
      "createdAt": "2026-03-04T10:00:00"
    },
    {
      "id": 102,
      "serviceId": 42,
      "imageUrl": "https://cdn.moer.kr/services/42/images/def456.webp",
      "thumbnailUrl": "https://cdn.moer.kr/services/42/images/def456_thumb.webp",
      "sortOrder": 2,
      "caption": null,
      "createdAt": "2026-03-04T10:05:00"
    }
  ]
}
```

**예외 처리**

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 404 | SV001 | 서비스를 찾을 수 없습니다. | serviceId 없음 |

**기타 참고사항**
- Public 예약 화면에서도 조회 가능해야 하므로, 별도 Public 엔드포인트도 필요
- Public 경로: `GET /api/public/businesses/{slug}/services/{serviceId}/images`

```json
{
  "success": true,
  "data": null
}
```

**예외 처리**

|-----------|-----------|-------------|------|

#### 1-4. 서비스 이미지 순서 변경

**기능명**: 서비스 이미지 순서 변경

**기능 설명**: 서비스에 등록된 이미지들의 표시 순서를 일괄 변경합니다. 드래그앤드롭으로 순서 변경 후 저장할 때 호출합니다.

**입력값**

- Method: `PATCH`
- Path: `/api/businesses/{businessId}/services/{serviceId}/images/sort`
- Path Variables:
  - `businessId` (Long, 필수)
  - `serviceId` (Long, 필수)
- Headers: `Authorization: Bearer {accessToken}`
- Request Body:
```json
{
  "imageOrders": [
    { "imageId": 102, "sortOrder": 1 },
    { "imageId": 101, "sortOrder": 2 }
  ]
}
```

**출력값**

```json
{
  "success": true,
  "data": null
}
```

**예외 처리**

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 400 | IMG005 | 이미지 ID 목록이 서비스의 이미지와 일치하지 않습니다. | imageId 불일치 |
| 403 | B001 | 해당 매장에 대한 권한이 없습니다. | 권한 없음 |

### 2. 리뷰 이미지 업로드 API (개선)

**기능명**: 리뷰 생성 시 이미지 동시 업로드

**기능 설명**: 현재 구현은 리뷰 생성 후 별도 API로 이미지를 업로드합니다(`POST /customer/reviews/{reviewId}/images`). 사용자 경험 개선을 위해 리뷰 생성 요청에 이미지를 multipart로 함께 전송하거나, 임시 업로드(presigned URL) 방식을 지원해야 합니다.

**방안 A: 리뷰 생성 API를 multipart로 변경**

**입력값**

- Method: `POST`
- Path: `/api/customer/businesses/{slug}/reviews`
- Headers:
  - `Authorization: Bearer {customerAccessToken}`
  - `Content-Type: multipart/form-data`
- Request Body (multipart/form-data):
  - `rating` (Integer, 필수): 1~5
  - `content` (String, 필수): 리뷰 내용 (최대 1000자)
  - `reservationNumber` (String, 필수): 예약 번호
  - `images[]` (MultipartFile[], 선택): 이미지 파일 배열 (최대 3장)

**출력값**

```json
{
  "success": true,
  "data": {
    "id": 55,
    "rating": 5,
    "content": "시술이 너무 마음에 들어요!",
    "images": [
      {
        "id": 201,
        "imageUrl": "https://cdn.moer.kr/reviews/55/images/img1.webp",
        "thumbnailUrl": "https://cdn.moer.kr/reviews/55/images/img1_thumb.webp",
        "sortOrder": 1
      }
    ],
    "createdAt": "2026-03-04T15:30:00"
  }
}
```

**방안 B: 임시 이미지 업로드 후 리뷰 생성 시 연결 (권장)**

1단계: 임시 이미지 업로드
- Method: `POST`
- Path: `/api/customer/reviews/temp-images`
- Response: `{ "tempImageId": "temp_abc123", "imageUrl": "...", "expiresAt": "..." }`

2단계: 리뷰 생성 시 tempImageId 참조
```json
{
  "rating": 5,
  "content": "시술이 마음에 들어요",
  "reservationNumber": "RS2026030400001",
  "tempImageIds": ["temp_abc123", "temp_def456"]
}
```

**예외 처리**

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 400 | IMG001 | 지원하지 않는 파일 형식입니다. | 파일 형식 오류 |
| 400 | IMG002 | 파일 크기가 초과되었습니다. (최대 10MB) | 파일 용량 초과 |
| 400 | IMG003 | 리뷰 이미지는 최대 3장까지 등록 가능합니다. | 이미지 최대 개수 초과 |
| 400 | RV001 | 이미 리뷰를 작성한 예약입니다. | 중복 리뷰 |
| 403 | CP002 | 고객 권한이 필요합니다. | 비로그인 또는 관리자 토큰 |
| 404 | CP003 | 예약을 찾을 수 없습니다. | 예약 없음 |
| 404 | IMG006 | 임시 이미지를 찾을 수 없거나 만료되었습니다. | 방안 B 사용 시 |

**기타 참고사항**
- 방안 A는 구현이 단순하지만, 네트워크 오류 시 재업로드 어려움
- 방안 B는 UX가 우수하나 임시 저장 로직 필요 (임시 이미지 TTL: 1시간 권장)
- 프론트엔드 호출 위치: `src/pages/booking/[slug]/review.vue`

### 3. FREE 플랜 기본 통계 API

**기능명**: FREE 플랜 대시보드 기본 통계 제공

**기능 설명**: 현재 통계 API(`/businesses/{businessId}/statistics/*`)는 PAID 플랜에서만 접근 가능합니다. FREE 플랜 사용자도 오늘/이번 주의 기본 통계(예약 수, 매출액)를 확인할 수 있도록 별도 기본 통계 엔드포인트를 제공합니다. 상세 통계(월간, 직원별, 서비스별)는 여전히 PAID 전용으로 유지합니다.

**입력값**

- Method: `GET`
- Path: `/api/businesses/{businessId}/dashboard/basic-stats`
- Path Variables:
  - `businessId` (Long, 필수)
- Headers: `Authorization: Bearer {accessToken}`
- 플랜 제한: FREE 포함 모든 플랜 허용

**출력값**

```json
{
  "success": true,
  "data": {
    "today": {
      "reservationCount": 8,
      "completedCount": 3,
      "cancelledCount": 1,
      "revenue": 120000
    },
    "thisWeek": {
      "reservationCount": 34,
      "completedCount": 28,
      "cancelledCount": 3,
      "revenue": 520000
    },
    "pendingReservations": 4,
    "unreadReviews": 2,
    "generatedAt": "2026-03-04T10:00:00"
  }
}
```

**예외 처리**

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 401 | A001 | 인증이 필요합니다. | 비로그인 |
| 403 | B001 | 해당 매장에 대한 권한이 없습니다. | 타 매장 접근 |

**기타 참고사항**
- 기존 `/dashboard/stats` API는 PAID 전용 유지, 이 API는 모든 플랜 허용
- 기존 대시보드 API와의 차이: 기본 통계 API는 오늘/이번 주 집계만 반환, 상세 그래프 데이터 미포함
- 프론트엔드 호출 위치: `src/pages/shop-admin/dashboard.vue`, `src/stores/dashboard.js`
- 프론트엔드 수정 계획: 통계 페이지에서 FREE 플랜일 때 전면 블러 대신 기본 통계는 표시하고, 상세 분석 영역만 블러 + 업그레이드 CTA

### 4. 인앱 도움말 콘텐츠 API

**기능명**: 인앱 도움말/FAQ 콘텐츠 조회

**기능 설명**: 관리자 패널 내에서 기능 설명, 사용 방법, FAQ를 조회할 수 있도록 도움말 콘텐츠를 제공합니다. 백엔드에서 관리하면 배포 없이 콘텐츠를 수정할 수 있습니다.

**판단 사항**: 트래픽이 낮고 내용 변경이 드문 경우 프론트엔드에서 정적 JSON 파일로 처리 가능. 다국어 지원, 빈번한 업데이트가 예상되는 경우 백엔드 API 권장.

**입력값**

- Method: `GET`
- Path: `/api/public/help`
- Query Parameters:
  - `category` (String, 선택): 도움말 카테고리 필터 (예: `reservation`, `staff`, `service`, `payment`, `statistics`)
  - `keyword` (String, 선택): 검색어
  - `lang` (String, 선택): 언어 코드 (기본값: `ko`)

**출력값**

```json
{
  "success": true,
  "data": {
    "categories": [
      {
        "id": "reservation",
        "name": "예약 관리",
        "icon": "mdi-calendar"
      },
      {
        "id": "staff",
        "name": "직원 관리",
        "icon": "mdi-account-group"
      }
    ],
    "items": [
      {
        "id": 1,
        "category": "reservation",
        "title": "예약을 확정하려면 어떻게 하나요?",
        "content": "예약 목록에서 예약 행의 상태를 '확정'으로 변경하거나, 예약 상세에서 '확정' 버튼을 클릭합니다.",
        "relatedFeature": "reservations/list",
        "sortOrder": 1,
        "updatedAt": "2026-02-01T00:00:00"
      }
    ],
    "totalCount": 42
  }
}
```

**예외 처리**

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 404 | PUB001 | 도움말 콘텐츠를 찾을 수 없습니다. | 카테고리 또는 항목 없음 |

**기타 참고사항**
- Public API로 인증 불필요 (로그인 전 도움말 조회 가능)
- 슈퍼관리자 전용 도움말 콘텐츠 관리 API(CRUD)를 `/api/superadmin/help` 하위에 별도 제공 권장
- 프론트엔드에서는 `?` 아이콘 클릭 시 해당 기능의 `relatedFeature` 코드로 필터링하여 표시 예정

### 5. 개별 직원 일일/주간 스케줄 조회 API

**기능명**: 직원별 예약 스케줄 뷰 조회

**기능 설명**: 캘린더 화면에서 특정 직원을 선택하면, 해당 직원의 예약만 날짜 범위로 조회합니다. 기존 `GET /staffs/{staffId}/reservations`는 단일 날짜만 지원하므로, 주간 범위 조회와 직원 가용 슬롯 정보를 포함한 확장 API가 필요합니다.

**입력값**

- Method: `GET`
- Path: `/api/businesses/{businessId}/staffs/{staffId}/schedule`
- Path Variables:
  - `businessId` (Long, 필수)
  - `staffId` (Long, 필수)
- Headers: `Authorization: Bearer {accessToken}`
- Query Parameters:
  - `startDate` (String, 필수): 조회 시작일 (yyyy-MM-dd)
  - `endDate` (String, 필수): 조회 종료일 (yyyy-MM-dd, 최대 31일)

### 7. 관리자에서 고객으로 알림 발송 API

**기능명**: 관리자 주도 고객 알림 발송

**기능 설명**: 관리자가 특정 고객 또는 고객 그룹(세그먼트)에게 예약 리마인더, 재방문 쿠폰, 프로모션 메시지 등을 직접 발송합니다.

**입력값**

- Method: `POST`
- Path: `/api/businesses/{businessId}/notifications/send`
- Path Variables:
  - `businessId` (Long, 필수)
- Headers: `Authorization: Bearer {accessToken}`
- Request Body:
```json
{
  "targetType": "SEGMENT",
  "targetIds": null,
  "segment": "INACTIVE",
  "notificationType": "PROMOTION",

  "message": "30일 이상 방문이 없으신 고객님께 15% 할인 쿠폰을 드립니다.",
  "channels": ["APP_PUSH", "KAKAO_ALIMTALK"],
  "scheduledAt": null
}
```

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| targetType | String | 필수 | `ALL` / `SPECIFIC` / `SEGMENT` |
| targetIds | Long[] | 조건부 | targetType이 SPECIFIC일 때 고객 ID 배열 |
| segment | String | 조건부 | targetType이 SEGMENT일 때: `ALL`, `VIP`, `REGULAR`, `NEW`, `INACTIVE`, `BIRTHDAY` |
| notificationType | String | 필수 | `REMINDER` / `PROMOTION` / `NOTICE` / `CUSTOM` |
| title | String | 필수 | 알림 제목 (최대 50자) |
| message | String | 필수 | 알림 내용 (최대 500자) |
| channels | String[] | 필수 | 발송 채널: `APP_PUSH`, `KAKAO_ALIMTALK`, `SMS` |
| scheduledAt | DateTime | 선택 | 예약 발송 시각 (null이면 즉시 발송) |

**출력값**

    "targetCount": 47,

| 400 | NTF001 | 발송 대상이 없습니다. | targetCount가 0 |
| 400 | NTF002 | 유효하지 않은 발송 채널입니다. | channels 값 오류 |
| 403 | SL003 | 이번 달 알림 발송 한도를 초과했습니다. | 플랜별 발송 한도 |
| 403 | TR002 | FREE 플랜에서는 사용할 수 없는 기능입니다. | 플랜 제한 |

**기타 참고사항**
- 발송 이력은 기존 `GET /businesses/{businessId}/notification-logs` API로 조회
- 발송 한도 예시: FREE 0회/월, BASIC 500회/월, PREMIUM 무제한
- 프론트엔드 호출 위치: `src/pages/shop-admin/customers/list.vue` (대량 발송), 고객 상세 화면(개별 발송)

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 400 | CRM004 | 주 고객 ID가 병합 목록에 포함될 수 없습니다. | primaryId 중복 포함 |
| 400 | CRM005 | 병합할 고객이 없습니다. | mergeCustomerIds 비어있음 |
| 404 | C001 | 고객을 찾을 수 없습니다. | customerId 없음 |

### 10. 랜딩 페이지 동적 통계 API

**기능명**: 서비스 이용 현황 통계 조회 (Public)

**기능 설명**: 랜딩 페이지의 히어로/통계 섹션에 표시되는 "가입 매장 수", "처리된 예약 수", "누적 리뷰 수" 등의 수치를 실제 데이터 기반으로 동적으로 표시합니다. 현재는 하드코딩된 수치를 사용 중입니다.

**입력값**

- Method: `GET`
- Path: `/api/public/platform-stats`
- Headers: 인증 불필요 (Public API)
- Query Parameters: 없음

**출력값**

```json
{
  "success": true,
  "data": {
    "totalBusinesses": 1240,
    "totalReservations": 58000,
    "totalReviews": 12400,
    "avgRating": 4.7,
    "activePlansCount": 890,
    "updatedAt": "2026-03-04T00:00:00"
  }
}
```

**예외 처리**

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 503 | STAT001 | 통계 데이터를 일시적으로 사용할 수 없습니다. | DB 오류 등 |

**기타 참고사항**
- 응답 캐싱 권장: TTL 1시간 (수치 실시간성 불필요)
- 민감한 정보 제외 (매출액, 사용자 개인정보 등)
- 프론트엔드 호출 위치: `src/components/public/landing/StatsSection.vue`, `HeroSection.vue`

### 11. 회원가입 플로우 변경

**기능명**: 회원가입 시 30일 무료 체험 자동 적용

**기능 설명**: 현재 회원가입 API(`POST /auth/register`)는 `plan` 필드를 포함하여, 가입 시 플랜을 선택해야 합니다. UX 개선을 위해 가입 시 플랜 선택을 제거하고, 가입과 동시에 30일 무료 체험이 자동으로 활성화되도록 변경합니다.

**현재 Request Body** (변경 전):
```json
{
  "email": "user@example.com",
  "password": "password123",
  "businessName": "내 네일샵",
  "businessType": "NAIL",
  "plan": "BASIC",
  "billingCycle": "MONTHLY"
}
```

**변경 후 Request Body**:
```json
{
  "email": "user@example.com",
  "password": "password123",
  "businessName": "내 네일샵",
  "businessType": "NAIL"
}
```

**변경 후 Response**:
```json
{
  "success": true,
  "data": {
    "userId": 999,
    "businessId": 888,
    "email": "user@example.com",
    "businessName": "내 네일샵",
    "accessToken": "eyJ...",
    "refreshToken": "eyJ...",
    "subscription": {
      "plan": "TRIAL",
      "trialEndsAt": "2026-04-03T23:59:59",
      "trialDaysRemaining": 30
    }
  }
}
```

**예외 처리** (기존 유지):

| HTTP 상태 | 에러 코드 | 에러 메시지 | 상황 |
|-----------|-----------|-------------|------|
| 409 | U002 | 이미 사용 중인 이메일입니다. | 이메일 중복 |
| 400 | U001 | 유효하지 않은 이메일 형식입니다. | 이메일 형식 오류 |
| 400 | U003 | 비밀번호는 8자 이상이어야 합니다. | 비밀번호 형식 오류 |

**기타 참고사항**
- `plan`, `billingCycle` 필드는 하위 호환성을 위해 일정 기간 수신 가능하나 무시 처리 권장
- 가입 완료 후 온보딩 페이지로 리다이렉트되어 매장 설정을 진행하는 기존 플로우 유지
- 프론트엔드 수정 위치: 회원가입 페이지에서 플랜 선택 UI 제거, 무료 체험 안내 문구 추가

### 12. SSE 실시간 알림 이벤트 타입 확장

**기능명**: SSE 실시간 알림 이벤트 추가

**기능 설명**: 현재 SSE 연결(`GET /notifications/stream` 또는 유사 엔드포인트)에서 지원하는 이벤트 타입이 제한적입니다. 프론트엔드의 `useSseNotifications.js`에서 처리할 수 있도록 이벤트 타입을 확장합니다.

**현재 지원 이벤트 (추정)**:
- `NEW_NOTIFICATION`: 일반 알림

**추가 요청 이벤트 타입**:

- SSE 연결 실패 시 프론트엔드에서 30초 간격 폴링으로 자동 폴백하는 기존 로직 유지
