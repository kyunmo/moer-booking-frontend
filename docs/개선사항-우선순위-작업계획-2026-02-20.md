## 2. P0: 즉시 수정 (1~3일)

> **기준:** 버그, 보안 이슈, 사용자 혼란을 즉시 유발하는 항목
> **예상 총 소요: ~12시간**

### P0-1. 가격 VAT 표시 통일
- **현황:** pricing.vue "월 20,000원", faq.vue "월 22,000원(VAT포함)", refund-policy "월 20,000원(VAT별도)" → 3가지 다른 표시
- **영향:** 신규 가입자(P4) 혼란, 신뢰도 하락
- **작업:** FE — 전 페이지 가격 표시를 "월 22,000원 (VAT 포함)" 또는 "월 20,000원 (VAT 별도)"로 통일
- **소요:** 1h
- **파일:** `pricing.vue`, `faq.vue`, `refund-policy.vue`, `constants/pricing.js`

### P0-2. NavBar 알림 더미 데이터 제거
- **현황:** Materio 영문 더미 데이터 6개가 실 서비스에서 표시
- **영향:** 매장 관리자(P2) 혼란, 서비스 신뢰도 하락
- **작업:** FE — NavBarNotifications 컴포넌트에서 더미 데이터 제거, 실제 API 연동 데이터만 표시
- **소요:** 1h
- **파일:** `layouts/components/NavBarNotifications.vue`

### P0-3. 미사용 패키지 제거 (mapbox-gl, jwt-decode)
- **현황:** mapbox-gl (~1MB+), jwt-decode 설치되어 있으나 미사용
- **영향:** 번들 크기 불필요 증가
- **작업:** FE — `npm uninstall mapbox-gl jwt-decode`
- **소요:** 0.5h
- **파일:** `package.json`

### P0-4. login.vue 테스트 계정 제거/환경변수 제어
- **현황:** 프로덕션 코드에 테스트 계정 노출 가능
- **영향:** 보안 위험
- **작업:** FE — VITE_TEST_EMAIL/PASSWORD 환경변수 있을 때만 표시, production 빌드에서 제거
- **소요:** 1h
- **파일:** `pages/login.vue`, `.env.production`

### P0-5. alert() → useSnackbar() 통일 (12곳)
- **현황:** 에러 처리에서 alert() 12곳 사용
- **영향:** 전체 UX 일관성 파괴, 모바일에서 alert 팝업 불편
- **작업:** FE — 모든 alert() 호출을 useSnackbar().error() 또는 warning()으로 교체
- **소요:** 2h
- **파일:** 전역 검색 `alert(` → 개별 교체

### P0-6. console.log/디버그 코드 제거
- **현황:** 프로덕션 코드에 디버그 출력 잔존
- **영향:** 성능 미세 영향, 프로페셔널리즘
- **작업:** FE — 전역 검색 후 제거 또는 vite 빌드에서 drop
- **소요:** 1.5h

### P0-7. nul 파일 삭제
- **현황:** Windows 환경에서 잘못 생성된 파일
- **작업:** FE — 파일 삭제
- **소요:** 0.1h

### P0-8. 포커스 표시(Focus Ring) 복구
- **현황:** CSS에서 `focus-visible::after { opacity: 0 !important }` → 키보드 사용자 포커스 위치 인식 불가
- **영향:** 접근성(A11y) 심각 위반
- **작업:** FE — 해당 override 제거 또는 적절한 focus-visible 스타일 추가
- **소요:** 1h
- **파일:** `@core/scss/base/_overrides.scss` 또는 관련 파일

### P0-9. Materio 미사용 다이얼로그 정리
- **현황:** 13개 미사용 다이얼로그 컴포넌트
- **영향:** 코드 혼란, 유지보수 부담
- **작업:** FE — 사용 여부 확인 후 미사용 파일 삭제
- **소요:** 2h
- **파일:** `src/components/dialogs/` (AddAuthenticatorAppDialog, AddEditAddressDialog, CardAddEditDialog 등)

## 3. P1: 베타 출시 전 필수 (1~2주)

> **기준:** 핵심 사용자 경험에 직접적 영향, 베타 테스터가 반드시 부딪힐 문제
> **예상 총 소요: ~40시간**

### P1-1. 예약 시간 충돌 검증
- **현황:** 같은 직원에게 같은 시간에 예약 2건 등록 가능
- **영향:** 매장 관리자(P2) 핵심 불편. 실 운영 시 큰 문제
- **작업:** FE+BE
  - BE: 예약 생성/수정 시 직원별 시간 중복 체크 API
  - FE: ReservationFormDialog에서 시간 선택 후 가용성 확인, 충돌 시 경고
- **소요:** 8h (FE 4h + BE 4h)
- **파일:** `ReservationFormDialog.vue`, `api/reservations.js`, `stores/reservation.js`

### P1-2. 모바일 테이블 → 카드 뷰 전환
- **현황:** VDataTable이 모바일에서 수평 스크롤만 가능
- **영향:** 매장 관리자(P2) 모바일 사용 불편
- **작업:** FE
  - `useDisplay()` composable로 `smAndDown` 감지
  - 예약 목록, 고객 목록에 모바일 카드 뷰 조건부 렌더링
- **소요:** 6h (예약 목록 3h + 고객 목록 3h)
- **파일:** `reservations/list.vue`, `customers/list.vue`

### P1-3. 신규 고객 프로필 등록 플로우 개선
- **현황:** OAuth 콜백 후 `isNewUser=true`인데 예약 페이지로 직접 갈 수 있음. Step 3에서야 전화번호 경고
- **영향:** 신규 고객(P1) 혼란, 예약 실패 가능
- **작업:** FE
  - oauth2-redirect에서 신규 고객 → 반드시 `/booking/profile` 리다이렉트
  - profile 페이지에 `redirectAfter` 쿼리 파라미터로 원래 목적지 보존
  - 전화번호 등록 완료 후 원래 예약 페이지로 자동 이동
- **소요:** 4h
- **파일:** `oauth2-redirect.vue`, `booking/profile.vue`, `stores/customer-auth.js`

### P1-4. 서비스 선택 → 예상 종료 시간 자동 계산
- **현황:** 서비스 다중 선택 후 총 소요 시간은 표시하나 종료 시간 미표시
- **영향:** 고객(P1, P3) + 관리자(P2) 모두 불편
- **작업:** FE
  - reserve.vue Step 2에서 시간 선택 후: "시작 14:00 → 종료 16:00 (커트 30분 + 펌 90분)"
  - ReservationFormDialog에서도 동일 적용
- **소요:** 3h
- **파일:** `booking/[slug]/reserve.vue`, `ReservationFormDialog.vue`

### P1-5. 예약 시간 오전/오후 표기 추가
- **현황:** "14:00" 24시간제만 표시
- **영향:** 고객(P1, P3) UX. 한국 사용자 대부분 오전/오후 선호
- **작업:** FE — 시간 포맷 유틸 함수 추가: `formatTimeKR("14:00") → "오후 2:00"`
  - 예약 시간 표시 전체에 적용
- **소요:** 2h
- **파일:** `utils/dateFormat.js` (신규), 관련 페이지 전체

### P1-6. ARIA 레이블 추가 (1차)
- **현황:** icon-only 버튼, bottom nav, 햄버거 메뉴 등 aria-label 90% 누락
- **영향:** 접근성(A11y) 기본 요구사항 미충족
- **작업:** FE — 주요 UI 요소에 aria-label 일괄 추가
  - BookingBottomNav 버튼
  - PublicHeader 햄버거/닫기 버튼
  - 모든 icon-only VBtn
  - VImg alt 텍스트
- **소요:** 4h
- **파일:** `BookingBottomNav.vue`, `PublicHeader.vue`, 전역 검색

### P1-7. OG 태그 및 SEO 메타 태그 추가
- **현황:** route meta에 title/description만 설정. og 태그, 구조화 데이터 없음
- **영향:** SNS 공유 시 미리보기 없음, 검색 노출 불리
- **작업:** FE
  - @unhead/vue 또는 vue-head 도입
  - 주요 퍼블릭 페이지에 og:title, og:description, og:image 추가
  - 매장 상세 페이지: 동적 og 태그 (매장명, 이미지)
- **소요:** 5h
- **파일:** `plugins/`, 주요 public 페이지

### P1-8. OAUTH_BASE_URL 등 코드 정리
- **현황:** 동일 URL 파싱 로직 3곳 중복, response.data || response 패턴 불일치, useRouter() store 내부 호출
- **영향:** 유지보수성, 잠재 버그
- **작업:** FE
  - OAUTH_BASE_URL → `utils/oauth.js` 추출
  - Store 내 useRouter() → router 인스턴스 직접 import
  - axios interceptor window.location.href → router.push
  - response.data || response 패턴 통일
- **소요:** 4h
- **파일:** `stores/auth.js`, `api/axios.js`, 관련 파일

## 4. P2: 베타 초기 개선 (2~4주)

> **기준:** 사용성 개선, 신뢰도 강화, 경험 품질 향상
> **예상 총 소요: ~30시간**

### P2-1. 예약 캘린더에 직원별 스케줄 표시
- **현황:** 캘린더에 전체 예약만 표시, 직원별 필터 없음
- **작업:** FE — FullCalendar에 직원 필터/리소스 뷰 추가
- **소요:** 5h

### P2-2. 다중 예약 일괄 상태 변경
- **현황:** 예약 1건씩만 상태 변경 가능
- **작업:** FE+BE — 체크박스 선택 + 일괄 "확정"/"완료" 액션
- **소요:** 4h (FE 2h + BE 2h)

### P2-3. 통계 필터 즉시 반영 + CSV 다운로드
- **현황:** "검색" 버튼 클릭 필요, 다운로드 기능 없음

### P2-4. 재예약 기능 (이전 서비스/스태프 자동 선택)
- **현황:** 매번 서비스/스태프 다시 선택해야 함
- **작업:** FE+BE — "같은 서비스로 재예약" 버튼 → 이전 예약 기반 자동 채움
- **소요:** 3h

### P2-5. 고객 후기/사례 섹션 추가 (랜딩)
- **현황:** 고객 후기 없음 → 신뢰도 부족
- **작업:** FE — TestimonialsSection 컴포넌트 추가, 최소 3개 후기
- **소요:** 3h

### P2-6. 매장 상세 페이지 개선
- **현황:** 서비스별 리뷰 필터 없음, 전화 걸기 링크 없음
- **작업:** FE — `tel:` 링크 추가, 리뷰에 서비스/스태프 필터 추가
- **소요:** 2h

### P2-7. 서비스 이름 중복 검증
- **현황:** 같은 이름의 서비스 2번 등록 가능
- **작업:** FE+BE — 등록/수정 시 동일 매장 내 이름 중복 체크
- **소요:** 2h

### P2-8. VIP/단골 임계값 커스터마이즈
- **현황:** VIP/단골 판정 기준 변경 불가
- **작업:** FE+BE — 매장 설정에 임계값 설정 UI 추가
- **소요:** 3h

### P2-9. 스킵 네비게이션 링크 추가
- **현황:** 키보드 사용자가 메뉴를 매번 탭해야 본문 접근
- **작업:** FE — `<a href="#main-content" class="skip-link">` 추가
- **소요:** 1h

### P2-10. 브레드크럼 네비게이션 추가
- **현황:** 관리자 페이지에 현재 위치 표시 없음
- **작업:** FE — 라우트 기반 자동 브레드크럼 컴포넌트
