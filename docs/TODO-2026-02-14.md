## 2. UX 사용성 평가 (페르소나 기법)

### 페르소나 A: 김미소 (소규모 미용실 원장, 42세)
> 디지털 중간 / 카카오톡은 잘 쓰지만 웹 서비스는 익숙하지 않음

**시나리오**: 랜딩 → 회원가입 → 매장설정 → 서비스/스태프 등록 → 일상 예약관리

| 여정 구간 | 현재 상태 | Pain Point |
|-----------|----------|------------|
| 랜딩 → 가입 | OK | - |
| 가입 직후 | **끊김** | 빈 대시보드만 보임, 뭘 해야 하는지 안내 없음 |
| 매장/서비스/스태프 등록 | OK (기능 존재) | 설정 메뉴를 직접 찾아가야 함 |
| 에러 발생 시 | **불편** | 브라우저 alert() 팝업 → 당황스러움 |
| "고객이 직접 예약" | **불가** | 고객 예약 페이지 자체 미존재 |
| 프로필/비밀번호 변경 | **불가** | 프로필 페이지 없음 (404) |

**필요 작업:**
- [ ] 온보딩 위저드 (가입 직후: 매장설정 → 서비스등록 → 스태프등록 → 완료)
- [ ] alert() → useSnackbar 전환 (12곳)
- [ ] 프로필 페이지 생성

### 페르소나 B: 이준혁 (필라테스 매니저, 28세)
> 디지털 높음 / SaaS 여러 개 사용 경험

**시나리오**: 기능 비교 → 가입 → 강사/수업 등록 → 일정관리 → 통계 분석

| 여정 구간 | 현재 상태 | Pain Point |
|-----------|----------|------------|
| 기능 비교 | OK (features, pricing) | - |
| 캘린더 일정관리 | **불편** | 10명 강사 스케줄 구분 불가 (스태프 필터 없음) |
| 통계 분석 | **부분** | 전월 대비 비교 불가, 기간 선택 없음 |
| 반복 예약 | **불가** | 매주 같은 시간 수업을 매번 수동 등록 |
| 영업시간 | **하드코딩** | 캘린더 businessHours 10:00-20:00 고정, 매장설정 미반영 |
| 데이터 내보내기 | **불가** | PRO 플랜에서 홍보하지만 미구현 |

**필요 작업:**
- [ ] 캘린더 스태프별 필터 추가
- [ ] 캘린더 영업시간을 매장설정과 연동
- [ ] 대시보드/통계에 기간 선택 기능

### 페르소나 C: 박서연 (고객/소비자, 35세)
> 디지털 높음 / 카카오톡으로 예약하고 싶음

**시나리오**: 매장 검색 → 스태프/포트폴리오 확인 → 예약 → 리마인더 → 리뷰

| 여정 구간 | 현재 상태 | Pain Point |
|-----------|----------|------------|
| 매장 검색 | **불가** | 고객용 페이지 자체 없음 |
| 스태프/포트폴리오 확인 | **불가** | 포트폴리오 미구현, 고객 공개 페이지 없음 |
| 예약 | **불가** | 고객 예약 페이지 미존재 |
| 리마인더 | **불가** | 카카오톡 알림 미연동 |
| 리뷰 | **불가** | 리뷰 시스템 전체 미구현 |

**필요 작업:**
- [ ] 고객용 예약 페이지 (`/booking/:businessSlug`)
- [ ] 카카오톡 알림 연동
- [ ] 리뷰 시스템

### 전반적 UX 이슈

| 이슈 | 현재 상태 | 개선 방향 |
|------|----------|----------|
| 에러 처리 | alert() 12곳 / useSnackbar 11곳 / VAlert 3곳 혼재 | useSnackbar로 통일 |
| NavBar 알림 | Materio 영문 더미 데이터 6개 하드코딩 | 실제 알림 연동 또는 제거 |

| 접근성 | 아이콘 버튼에 aria-label 없음, 색상만으로 상태 구분 | aria-label 추가 |
