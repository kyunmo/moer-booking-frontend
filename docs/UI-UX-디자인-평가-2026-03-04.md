## 1. 시각적 디자인 평가

### 1.1 색상 체계

**긍정적 요소**:
- Vuetify semantic color 토큰 (`rgb(var(--v-theme-primary))`)을 전역에서 일관되게 사용
- 상태별 색상 코딩이 모든 페이지에서 통일: `warning`=대기, `primary`=확정, `success`=완료, `error`=취소
- CSS 변수 기반으로 다크/라이트 테마 전환이 안정적

**문제점 1** - `PublicFooter.vue`에 하드코딩 + CSS 미지원 함수 혼재:
```scss
// 하드코딩 색상과 CSS 변수 중복
background-color: #1e1b2e;
background-color: rgb(var(--v-theme-surface-variant), 1);

// color-mix()는 Safari 16.2 미만 미지원
background-color: color-mix(in srgb, rgb(var(--v-theme-surface-variant)) 85%, black);
```
> 소상공인 사용자 중 구형 iOS 기기 사용자가 있을 경우 배경색이 깨질 수 있습니다.

**문제점 2** - `statistics/index.vue` blur 오버레이의 pointer-events 미차단:
```css
.statistics-blurred {
  filter: blur(6px);
  /* pointer-events: none; 누락 → 클릭 가능 */
}
```

### 1.2 타이포그래피

**문제점 1** - `dashboard.vue`에서 시맨틱 불일치:
```html
<!-- subtitle 컨테이너에 헤딩 크기 적용 -->
<VListItemSubtitle class="text-h6">{{ stats.monthStats.totalReservations }}건</VListItemSubtitle>
```

**문제점 2** - h1 태그 3개 중복 (가장 심각한 접근성 문제):
```html
<!-- PublicHeader.vue -->
<h1 class="app-logo-title ms-2">{{ themeConfig.app.title }}</h1>

<!-- HeroSection.vue -->
<h1 class="hero-title mb-4">예약 관리, 이제 자동으로 해결하세요</h1>

<!-- PublicFooter.vue -->
<h1 class="footer-title">모에르(MOER)</h1>
```
> 한 페이지에 h1이 3개입니다. 로고와 푸터의 h1을 `span` 또는 `div`로 교체해야 합니다.

### 1.3 반응형 디자인

**문제점** - `services/list.vue` 헤더에 컨트롤 4개 인라인 배치:
```html
<VCardTitle class="d-flex align-center pe-2">
  <VSelect style="max-inline-size: 200px;" />   <!-- 카테고리 필터 -->
  <VTextField style="max-inline-size: 250px;" /> <!-- 검색 -->
  <VBtn>카테고리 관리</VBtn>
  <VBtn>서비스 등록</VBtn>
</VCardTitle>
```
> 1024px 이하 화면에서 overflow 및 레이아웃 붕괴 발생. 필터 영역을 `VCardText`로 분리해야 합니다.

## 7. 접근성(A11y) 평가

### 잘 된 점
- Vuetify 컴포넌트가 기본적인 ARIA 속성 제공
- 색상 대비가 Vuetify 기본 테마로 대체로 적합
- 폼 입력에 label 속성 일관 적용

### 문제점
| 항목 | 상태 | 권장 |
|------|------|------|
| h1 중복 | 한 페이지 3개 | 1개만 유지 |

| 포커스 관리 | 다이얼로그 열림/닫힘 시 포커스 트래핑 미확인 | Vuetify 기본 동작 검증 필요 |
| alt 텍스트 | 이미지에 alt 텍스트 일부 누락 | 모든 img에 의미있는 alt 추가 |

## 9. 결론

모에르(MOER)는 Materio 템플릿 기반 컴포넌트 시스템을 잘 활용하고 있으며, 반응형 분기 처리, 상태 피드백, CRUD 패턴 일관성이 전반적으로 우수합니다.

### 3대 핵심 개선 영역

**1순위 - 접근성 버그 수정**: h1 태그 중복, pointer-events 미차단, 취소 처리 로딩 없음 등 즉시 수정 가능
