## 1. 디자인 시스템 평가

### 1.1 색상 시스템

#### 현재 구현
| 색상 | 값 | 용도 | 평가 |
|------|----|----|------|
| Primary | `#666CFF` | CTA 버튼, 링크, 활성 상태 | 일관됨 |
| Secondary | `#6D788D` | 보조 버튼, 비활성 요소 | 일관됨 |
| Success | `#72E128` | 완료 상태, 활성 표시 | 일관됨 |
| Warning | `#FDB528` | 주의 알림, 대기 상태 | 일관됨 |
| Error | `#FF4D49` | 삭제, 에러, 취소 | 일관됨 |
| Info | `#26C6F9` | 정보, 확정 상태 | 일관됨 |

#### WCAG 색상 대비 분석
| 조합 | 대비율 | AA (4.5:1) | AAA (7:1) |
|------|--------|-----------|-----------|
| Primary vs 흰 배경 | ~4.5:1 | 통과 (근소) | **미달** |
| Secondary vs 흰 배경 | ~7.5:1 | 통과 | 통과 |
| Error vs 흰 배경 | ~3.9:1 | **미달** | **미달** |
| 텍스트(#262B43) vs 배경(#F7F7F9) | ~13:1 | 통과 | 통과 |
| Dark 보조텍스트(#4A5072) vs Dark BG(#282A42) | ~5.3:1 | 통과 | **미달** |

#### 개선 필요
- Error 색상: `#FF4D49` → `#D32F2F` (대비율 6.8:1로 AA 안정적 통과)
- Primary 텍스트 사용 시 배경과 최소 4.5:1 보장

## 3. 접근성(A11y) 평가

### 3.1 현재 접근성 수준 (WCAG 2.1 기준)

| 원칙 | 수준 | 평가 | 근거 |
|------|------|------|------|
| **인식 가능** (Perceivable) | A | **부분 충족** | 이미지 alt 텍스트 대부분 누락, 색상 대비 부분 미달 |
| **운용 가능** (Operable) | A | **부분 충족** | 키보드 네비게이션 기본 지원, 포커스 표시 제거됨 |
| **이해 가능** (Understandable) | A | **충족** | 폼 레이블 존재, 에러 메시지 제공 |
| **견고함** (Robust) | A | **충족** | 시맨틱 HTML, Vuetify ARIA 기본 지원 |

### 3.2 심각한 접근성 이슈

#### 이슈 1: 포커스 표시(Focus Ring) 제거 — 심각도: 높음
```scss
// _overrides.scss에서 발견
.v-btn:focus-visible::after {
  opacity: 0 !important;  // 포커스 아웃라인 완전 제거
}
```
**영향:** 키보드 사용자가 현재 포커스 위치를 알 수 없음
**개선:**
```scss
.v-btn:focus-visible {
  outline: 2px solid rgb(var(--v-theme-primary));
  outline-offset: 2px;
}
```

#### 이슈 2: ARIA 레이블 대규모 누락 — 심각도: 높음
| 요소 | 현황 | 예시 |
|------|------|------|
| Icon-only 버튼 | 90% 누락 | 햄버거 메뉴, 닫기, 검색 |
| Bottom Nav 버튼 | 누락 | 검색, 내예약, 프로필 |
| 이미지 alt | 대부분 누락 | 매장 프로필, 포트폴리오 |
| Dialog | aria-modal 부재 | 삭제 확인, 예약 상세 |

#### 이슈 3: 색상만으로 상태 구분 — 심각도: 중간
```
예약 상태: PENDING(주황), CONFIRMED(파랑), COMPLETED(초록), CANCELLED(빨강)
→ 색각이상 사용자가 구분 어려울 수 있음
→ 아이콘이나 텍스트로 보조 필요
```
**현재:** VChip에 색상 + 텍스트 함께 사용 → 부분 대응됨 (양호)

#### 이슈 4: 스킵 네비게이션 링크 없음 — 심각도: 중간
- 키보드 사용자가 헤더/네비게이션을 매번 탭해야 본문 접근
- `<a href="#main-content" class="skip-link">본문 바로가기</a>` 추가 필요

### 3.3 양호한 접근성 구현

- **폼 레이블:** VTextField의 `label` prop 사용 → 적절
- **에러 메시지:** 폼 유효성 검사 메시지 제공
- **시맨틱 HTML:** Vuetify 컴포넌트가 적절한 role 속성 자동 생성
- **다크 모드:** 약한 시력 사용자를 위한 대안 제공
- **예약 상태 칩:** 색상 + 텍스트 병행 → 색각이상 대응

### 3.4 접근성 점수: 5/10

## 4. UX 패턴 일관성 평가

### 4.1 다이얼로그 패턴

#### 표준화된 패턴 (일관)
```vue
<SomeDialog
  v-model="showDialog"      // Boolean
  :loading="isLoading"      // 로딩 상태
  @confirm="handleConfirm"  // 확인 이벤트
/>
```
- 모든 다이얼로그가 `modelValue` prop + `update:modelValue` emit 패턴 사용: **우수**
- ConfirmDeleteDialog 공통 컴포넌트 제공: **우수**

#### 불일관 패턴
- Materio 원본 vs 프로젝트 표준 ConfirmDialog **2종 혼재** (F-7)
- 일부 다이얼로그가 VDialog 직접 사용, 일부는 래퍼 컴포넌트 사용

### 4.2 로딩 상태 처리

#### 3계층 로딩 패턴 (일관)
| 레벨 | 구현 | 평가 |
|------|------|------|
| 전역 (라우트 전환) | AppLoadingIndicator (Suspense) | 우수 |
| 페이지 (데이터 로드) | VProgressCircular + VSkeletonLoader | 우수 |
| 컴포넌트 (폼 제출) | VBtn :loading | 우수 |

#### 불일관 패턴
- 일부 페이지: `v-if="loading"` → VProgressCircular
- 다른 페이지: VSkeletonLoader
- **통일된 기준 없음** (어떤 경우에 어떤 로딩을 사용하는지)

### 4.3 에러 처리 패턴

#### 구조화된 에러 흐름 (일관)
```
HTTP 요청 → Axios 인터셉터 → 에러코드 분기 → useSnackbar().error()
```
- 401: 토큰 갱신 시도 → 실패 시 로그아웃
- 402/403: Trial/Feature 제한 다이얼로그
- 기타: snackbar 에러 메시지

#### 불일관 패턴
| 패턴 | 사용처 | 문제 |
|------|--------|------|
| `useSnackbar().error()` | 대부분 | 일관됨 (표준) |
| `alert()` | 12곳 | **비표준**, UX 단절 |
| `console.log` 에러 | 여러 곳 | 사용자에게 피드백 없음 |
| 빈 catch 블록 | Store 대부분 | 에러 추적 불가 |

### 4.4 알림/토스트 메시지

#### 표준 패턴 (일관)
```javascript
const { success, error, warning, info } = useSnackbar()
success('저장되었습니다')   // 3초, 초록
error('오류가 발생했습니다') // 3초, 빨강
```

#### 메시지 스타일 가이드 부재
- 성공: "저장되었습니다", "등록되었습니다", "수정되었습니다" → 일관
- 에러: 백엔드 에러 메시지 그대로 표시 (한국어 번역됨) → 일관
- **경고/정보 메시지 사용 빈도 낮음**

### 4.5 네비게이션 패턴

| 패턴 | 구현 | 평가 |
|------|------|------|
| 선언적 라우팅 | RouterLink, VBtn :to | 우수 |
| 프로그래밍적 | router.push(), router.replace() | 적절 |
| 로그인 후 리다이렉트 | `?redirect=` 쿼리 파라미터 | 우수 |
| 브레드크럼 | **미구현** | 개선 필요 |

### 4.6 빈 상태(Empty State) 처리

- EmptyState 공통 컴포넌트 제공: **우수**
```vue
<EmptyState
  icon="ri-inbox-line"
  title="등록된 서비스가 없습니다"
  description="서비스를 등록하여 고객이 예약할 수 있도록 해보세요"
  action-label="서비스 등록"
  @action="openCreateDialog"
/>
```
- 대부분의 목록 페이지에서 빈 상태 처리: **양호**

### 4.7 일관성 점수: 7.5/10

### 8.2 SWOT 분석

#### Strengths (강점)
- 완벽한 12-column 반응형 그리드 활용
- 3계층 로딩 상태 패턴 (전역/페이지/컴포넌트)
- 다크/라이트 모드 완전 지원
- 구조화된 에러 처리 (에러코드 → 다이얼로그/스낵바)
- 표준화된 Dialog v-model 패턴
- EmptyState 공통 컴포넌트
- Trial/Feature Lock 시스템 완성도

#### Weaknesses (약점)
- 접근성 속성 대규모 누락 (ARIA, 포커스)
- VDataTable 모바일 대응 미흡
- alert() 잔존 (12곳)
- 미사용 패키지/코드 (mapbox-gl, Materio 잔재)
- SEO/OG 태그 미구현
- 가격 VAT 표시 불일치

#### Opportunities (기회)
- Vuetify useDisplay() 활용 → 테이블/카드 뷰 전환
- @unhead/vue 도입 → SEO 개선
- 접근성 개선 → 공공기관/기업 고객 확대
- PWA 적용 → 설치형 앱 경험

#### Threats (위협)
- 접근성 법적 요구사항 증가 추세
- Materio 레거시 코드 유지보수 부담
- 경쟁사(네이버 예약) UI/UX 수준 높음
