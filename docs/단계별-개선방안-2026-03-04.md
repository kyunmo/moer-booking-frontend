## Phase 1: 즉시 수정 (버그/접근성 - 1주)

> **목표**: 코드 품질 및 접근성 기본기 확보. 최소 노력으로 최대 효과.

### 1.1 h1 태그 중복 수정
- **파일**: `PublicHeader.vue`, `PublicFooter.vue`
- **작업**: 로고/푸터의 `<h1>`을 `<span>` 또는 `<div>`로 교체
- **이유**: 한 페이지에 h1이 3개 → SEO 패널티 + 스크린리더 혼란
- **난이도**: 낮음 (5분)

### 1.2 통계 blur 영역 인터랙션 차단
- **파일**: `statistics/index.vue`
- **작업**: `.statistics-blurred`에 `pointer-events: none; user-select: none;` 추가
- **이유**: 블러 처리된 차트가 클릭 가능하여 의도치 않은 인터랙션 발생
- **난이도**: 낮음 (5분)

### 1.3 예약 취소 로딩 표시 추가
- **파일**: `reservations/list.vue`
- **작업**: `cancelLoading` ref 추가, 취소 VBtn에 `:loading="cancelLoading"` 바인딩
- **이유**: 취소 버튼 클릭 후 응답까지 사용자 피드백 없음
- **난이도**: 낮음 (15분)

### 1.4 CSS 호환성 수정
- **파일**: `PublicFooter.vue`
- **작업**: `color-mix()` 앞에 fallback 색상값 추가
- **이유**: Safari 16.2 미만에서 배경색 깨짐 가능
- **난이도**: 낮음 (5분)

**Phase 1 예상 효과**: 접근성 점수 72 → 80, 기본 품질 확보

## Phase 2: 핵심 UX 개선 (2-4주)

> **목표**: 사용자 핵심 여정(예약 플로우, 대시보드)의 경험 향상

### 2.1 대시보드 정보 재배열
- **파일**: `dashboard.vue`
- **현재**: 9개 영역이 동등 비중으로 나열
- **변경**:
  ```
  1. 오늘 예약 목록 (운영 핵심) ← 최상단으로 이동
  2. 처리 필요 알림 (미할당 예약, 미답변 리뷰)
  3. 오늘 통계 카드 4개
  4. 주간 차트 + 이달 요약
  5. 퀵 액션 4개
  6. 신규 고객 목록
  7. 환영 배너 / 트라이얼 배너 (접히는 형태)
  ```
- **이유**: 관리자가 앱을 열면 가장 먼저 "오늘 무엇을 해야 하는지" 알아야 함
- **난이도**: 중간 (섹션 순서 변경 + 접히는 배너 처리)

### 2.2 서비스 목록 헤더 레이아웃 수정
- **파일**: `services/list.vue`
- **현재**: VCardTitle에 필터 4개 인라인 → 1024px 이하에서 overflow
- **변경**: 다른 목록 페이지(customers/list, reservations/list)처럼 필터를 VCardText 영역으로 분리
- **난이도**: 낮음~중간

### 2.3 예약 플로우 Step 2 레이아웃 개선
- **파일**: `reserve.vue`
- **현재**: 단일 컬럼 (달력 → 시간 → 스태프 → 다음) → 긴 스크롤
- **변경**:
  ```
  데스크톱 (md 이상):
  ┌─────────────┬─────────────┐
  │  VDatePicker │  시간 슬롯   │
  │             │  (오전/오후)  │
  │             │  스태프 선택  │
  └─────────────┴─────────────┘

  모바일 (sm 이하):
  기존 단일 컬럼 유지
  ```
- **이유**: 데스크톱에서 불필요한 스크롤 제거, 날짜-시간 동시 확인 가능
- **난이도**: 중간

### 2.4 매장 상세 Sticky CTA 버튼
- **파일**: `booking/[slug]/index.vue`
- **작업**: 페이지 하단에 Sticky "예약하기" 버튼 추가 (모바일에서 특히 중요)
- **이유**: 서비스 목록 스크롤 후 예약 시작점을 놓침
- **난이도**: 낮음

### 2.5 에러 복구 UI 추가
- **파일**: `reserve.vue`
- **작업**: 날짜/시간 로드 실패 시 "다시 시도" 버튼 표시
- **이유**: 네트워크 오류 시 toast만 표시되고 재시도 수단 없음
- **난이도**: 낮음

### 2.6 날짜 입력 VDatePicker 전환
- **파일**: `ReservationFormDialog.vue` 등 `type="date"` 사용 위치
- **작업**: 네이티브 date input → Vuetify VDatePicker 교체
- **이유**: 크로스 브라우저 UI 일관성 확보
- **난이도**: 낮음~중간

### 2.7 결제/구독 헤더 스타일 통일
- **파일**: `payment/index.vue`, `subscription/index.vue`
- **작업**: div 기반 헤더를 VCard 기반으로 통일 (다른 admin 페이지와 동일)
- **난이도**: 낮음

**Phase 2 예상 효과**: 전체 UX 71 → 78, 관리자 만족도 6.9 → 7.5

## Phase 3: 기능 보완 (1-2개월)

> **목표**: 사용자 불만이 높은 기능 부재 해결

### 3.1 서비스 이미지 업로드 기능
- **영향**: Persona B(네일아티스트)에게 가장 치명적 부재
- **작업**:
  1. ServiceFormDialog에 이미지 업로드 필드 추가 (최대 3장)
  2. 서비스 카드에 대표 이미지 썸네일 표시
  3. 고객 예약 페이지에서 서비스 이미지 표시
- **필요**: 백엔드 이미지 업로드 API 연동
- **난이도**: 높음

### 3.2 실시간 예약 알림
- **영향**: 모든 관리자 페르소나 (수동 새로고침 필요 제거)
- **작업**:
  1. SSE (Server-Sent Events) 연결 구현 (이미 `useSseNotifications.js` 존재)
  2. 새 예약, 취소, 리뷰 등 실시간 알림
  3. 브라우저 Push Notification 연동
- **난이도**: 중간 (SSE 인프라 기존 존재)

### 3.3 회원가입 플로우 간소화
- **영향**: Persona A, B (가입 진입 장벽)
- **작업**:
  1. 회원가입 시 플랜 선택 제거 → 기본 30일 무료 체험
  2. 소셜 로그인 옵션 추가 (로그인과 동일)
  3. 무료 체험 안내를 가입 페이지에 명확히 표시
- **난이도**: 중간

### 3.4 FREE 플랜 기본 통계 제공
- **영향**: Persona A, B (가장 높은 좌절감)
- **작업**:
  1. 오늘/이번주 기본 통계는 FREE에서도 제공
  2. 월간/직원별/서비스별 상세 통계만 PAID 전용
  3. 블러 대신 "더 자세한 통계를 보려면 업그레이드" CTA
- **이유**: 전면 블러는 사용자를 좌절시켜 이탈로 이어질 수 있음
- **난이도**: 중간

### 3.5 리뷰 작성 시 이미지 업로드
- **영향**: Persona C (고객)
- **작업**: 리뷰 작성 폼에 이미지 선택 UI 추가, 제출 시 함께 업로드
- **현재**: 리뷰 제출 후에만 이미지 추가 가능 (리뷰 ID 필요)
- **해결**: 이미지를 미리 선택해두고, 리뷰 생성 응답의 ID로 연이어 이미지 업로드
- **난이도**: 중간

**Phase 3 예상 효과**: 전체 UX 78 → 84, 프리랜서 만족도 6.6 → 7.5

### 4.3 하단 네비게이션 개선 (고객용)

- **효과**: 주요 기능에 대한 즉시 접근성 향상

### 4.4 인앱 도움말 시스템

  3. 복잡한 설정(직원 스케줄, 카테고리 등)에 안내 문구 삽입
- **난이도**: 중간

**Phase 4 예상 효과**: 전체 UX 84 → 88, IA 65 → 80, H10 55 → 75
