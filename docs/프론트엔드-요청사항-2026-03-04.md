## 1. 이미지 URL 형식 변경

### 서비스 이미지
백엔드는 로컬 파일 시스템에 저장하며, CDN URL이 아닌 로컬 경로를 반환합니다.

```
// 요구사항 문서의 URL
"imageUrl": "https://cdn.moer.kr/services/42/images/abc123.webp"

// 실제 반환 URL
"imageUrl": "/uploads/services/42/a1b2c3d4-5678-uuid.jpg"
```

**프론트엔드 처리:**
- `baseURL + imageUrl`로 조합하여 사용
- 예: `http://localhost:8080/uploads/services/42/uuid.jpg`
- 썸네일 URL(`thumbnailUrl`)은 현재 `null` (썸네일 자동 생성 미구현)
- 향후 CDN 도입 시 URL 형식만 변경 예정

### 리뷰 이미지 동일
```
"imageUrl": "/uploads/reviews/55/uuid.jpg"
```

## 2. 회원가입 Response 변경

### 변경 사항
- `plan`, `billingCycle` 파라미터를 보내도 **무시**됩니다
- 항상 FREE 플랜 + 30일 TRIAL로 자동 생성

### Request (변경 후)
```json
{
  "email": "user@example.com",
  "password": "password123",
  "name": "홍길동",
  "businessName": "내 네일샵",
  "businessType": "NAIL"
}
```

### Response
기존 `trial` 필드에 체험판 정보가 포함되어 있습니다:
```json
{
  "success": true,
  "data": {
    "accessToken": "eyJ...",
    "refreshToken": "eyJ...",
    "tokenType": "Bearer",
    "expiresIn": 3600,
    "user": { ... },
    "business": { ... },
    "trial": {
      "startedAt": "2026-03-04T10:00:00",
      "expiresAt": "2026-04-03T10:00:00",
      "remainingDays": 30,
      "isPremium": false
    }
  }
}
```

**프론트엔드 액션:**
- 회원가입 폼에서 플랜 선택 UI 제거
- "30일 무료 체험 시작" 안내 문구 표시
- `trial.remainingDays`로 체험 잔여일 표시

## 3. SSE 이벤트 타입 및 데이터 구조

### 새로 추가된 이벤트 타입

| 이벤트 | 발생 시점 | 대상 |
|--------|----------|------|
| `RESERVATION_CREATED` | 고객이 예약 생성 시 | 매장 owner |
| `RESERVATION_CANCELLED` | 고객이 예약 취소 시 | 매장 owner |
| `REVIEW_CREATED` | 고객이 리뷰 작성 시 | 매장 owner |
| `HEARTBEAT` | 30초마다 자동 | 모든 연결 |

### 이벤트 데이터 구조

```javascript
// RESERVATION_CREATED
{
  "type": "RESERVATION_CREATED",
  "referenceId": 301,
  "reservationNumber": "RS2026030400001",
  "customerName": "홍길동",
  "serviceName": "젤 네일",
  "startTime": "2026-03-05T14:00:00",
  "staffName": "김지수",
  "message": "새 예약이 들어왔습니다.",
  "createdAt": "2026-03-04T11:30:00"
}

// RESERVATION_CANCELLED
{
  "type": "RESERVATION_CANCELLED",
  "referenceId": 301,
  "reservationNumber": "RS2026030400001",
  "customerName": "홍길동",
  "startTime": "2026-03-05T14:00:00",
  "reason": null,
  "message": "예약이 취소되었습니다.",
  "createdAt": "2026-03-04T11:35:00"
}

// REVIEW_CREATED
{
  "type": "REVIEW_CREATED",
  "referenceId": 55,
  "customerName": "홍길동",
  "rating": 5,
  "contentPreview": "시술이 너무 마음에 들어요!",

// HEARTBEAT
{
  "type": "HEARTBEAT",
  "timestamp": "2026-03-04T11:40:00"

```

**프론트엔드 액션 (`useSseNotifications.js`):**
- 이벤트명으로 `addEventListener` 등록
- `HEARTBEAT` 수신 시 연결 상태 갱신 (30초 이상 미수신 시 재연결)

## 4. 리뷰 이미지 multipart 업로드

### 기존 JSON 방식 (유지)
```javascript
// Content-Type: application/json
POST /api/customer/businesses/{slug}/reviews
{ "rating": 5, "content": "...", "reservationNumber": "RS..." }
```

### 새 multipart 방식 (추가)
```javascript
// Content-Type: multipart/form-data
const formData = new FormData();
formData.append('rating', 5);
formData.append('content', '시술이 마음에 들어요');
formData.append('reservationNumber', 'RS2026030400001');
formData.append('images', file1);
formData.append('images', file2);

POST /api/customer/businesses/{slug}/reviews
```

**응답에 이미지 정보 포함:**
```json
{
  "success": true,
  "data": {
    "id": 55,
    "rating": 5,
    "content": "시술이 마음에 들어요",
    "images": [
      { "id": 201, "imageUrl": "/uploads/reviews/55/uuid.jpg", "sortOrder": 0 }
    ],
    ...
  }
}
```

## 5. 서비스 이미지 API

### 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/businesses/{bid}/services/{sid}/images` | 이미지 업로드 (multipart) |
| GET | `/api/businesses/{bid}/services/{sid}/images` | 이미지 목록 조회 |
| DELETE | `/api/businesses/{bid}/services/{sid}/images/{imageId}` | 이미지 삭제 |
| PATCH | `/api/businesses/{bid}/services/{sid}/images/sort` | 순서 변경 |
| GET | `/api/public/businesses/{slug}/services/{sid}/images` | Public 조회 |

### 업로드 (multipart/form-data)
```javascript
const formData = new FormData();
formData.append('file', imageFile);
formData.append('sortOrder', 1);       // 선택
formData.append('caption', '시술 결과'); // 선택
```

### 순서 변경
```json
POST /api/businesses/1/services/42/images/sort
{
  "imageOrders": [
    { "imageId": 102, "sortOrder": 1 },
    { "imageId": 101, "sortOrder": 2 }
  ]
}
```

## 7. 예약 Reschedule API (캘린더 드래그앤드롭)

```
PATCH /api/businesses/{bid}/reservations/{rid}/reschedule
{
  "newDate": "2026-03-05",
  "newStartTime": "14:00",
  "newEndTime": "15:30",     // 선택 (null이면 기존 소요시간 유지)
  "staffId": 7,              // 선택 (null이면 기존 직원)
  "notifyCustomer": true     // 선택 (기본 true)
}
```

**프론트엔드 액션 (FullCalendar):**
- `eventDrop` / `eventResize` 이벤트에서 호출
- 실패 시 `info.revert()` 호출하여 원위치

## 8. 직원 주간 스케줄 API

```
GET /api/businesses/{bid}/staffs/{sid}/schedule?startDate=2026-03-02&endDate=2026-03-08&view=week
```

응답에 `reservations[]`, `workSchedule[]`, `blockedSlots[]` 포함.

---

## 9. CSV 내보내기

```
GET /api/businesses/{bid}/customers/export?segment=VIP&encoding=UTF-8
```

- `window.location.href`로 직접 호출하여 파일 다운로드
- UTF-8 BOM 포함 (엑셀 한글 정상 표시)
- BASIC 플랜 이상에서만 사용 가능

## 미구현 사항 (백엔드 후속 작업 필요)

| 항목 | 설명 |
|------|------|
| 이미지 썸네일 자동 생성 | `thumbnailUrl`이 현재 null. 이미지 리사이즈 라이브러리 추가 필요 |
| 카카오 알림톡 실제 발송 | 설정 저장만 구현. 카카오 비즈 파트너 API 연동 별도 작업 |
| 알림 예약 발송 | `scheduledAt` 파라미터 수신 가능하나 즉시 발송만 처리 |
| 직원 blockedSlots | 현재 빈 배열 반환. 블록 슬롯 관리 기능 추가 필요 |
