## 2. 미완성 항목 상세 분석

### 2.1 🔴 출시 필수 (Must-Have for Beta)

#### A. 약관/개인정보처리방침 정식 버전
- **현황:** 임시(placeholder) 텍스트만 존재
- **위치:** `/terms`, `/privacy`
- **리스크:** 개인정보보호법 위반 가능, 서비스 출시 불가
- **예상 소요:** 4~6시간 (법률 검토 포함)
- **조언:**
  - 개인정보보호위원회 표준 개인정보처리방침 양식 활용
  - B2B2C 특성상 "예모가 매장에게 제공하는 서비스"와 "매장이 고객에게 제공하는 서비스" 구분 명확히
  - 제3자 제공(카카오 알림톡 등) 조항 포함
  - 환불 정책 페이지 (`/refund-policy`) 신규 작성 필요

#### B. 네이버 웍스 이메일 발송
- **현황:** UI 완성 (forgot-password.vue, reset-password.vue), 백엔드 미연동
- **영향 범위:** 비밀번호 찾기, 비밀번호 재설정
- **예상 소요:** 3~4시간
- **작업:**
  1. 네이버 웍스 SMTP 설정 (application.yml)
  2. Spring Boot `JavaMailSender` 설정
  3. 이메일 템플릿 HTML 작성 (비밀번호 재설정 링크)
  4. 발송 테스트
  5. 발송 실패 시 에러 핸들링

#### C. 슬러그(slug) 기반 예약 페이지 재검토
- **현황:** 구현 완료이나 검증 필요
- **확인 사항:**
  1. `GET /api/businesses/{businessId}` 응답에 `slug` 필드 포함 여부 → 이전에 누락 이슈 발견됨
  2. `/booking/{slug}` → 매장 상세 → 예약 전체 플로우 E2E 테스트
  3. 슬러그 미설정 매장의 디폴트 처리
  4. SEO 메타태그 (og:title, og:description 등)
  5. 슬러그 변경 시 기존 URL 리다이렉트 정책
- **예상 소요:** 2~3시간 (테스트 + 수정)

#### D. 관리자 프로필 페이지
- **현황:** 미구현 (404 발생), 라우팅에 `/shop-admin/profile` 없음
- **필요 기능:**
  - 기본 정보 표시/수정 (이름, 이메일, 전화번호)
  - 비밀번호 변경 (API: `auth.changePassword()` 이미 존재)
  - 회원 탈퇴 (확인 다이얼로그 + 데이터 삭제 안내)
  - SNS 계정 연결/해제 관리 (Phase 2)
- **예상 소요:** 4~5시간

## 4. 시니어 기획자 조언: 놓치기 쉬운 사항

### 4.1 🎯 베타 출시 전 필수 점검

1. **에러 상태 페이지** — 404, 500 에러 시 사용자에게 보여줄 페이지가 있는지 확인. 빈 화면 노출은 서비스 신뢰 하락.

2. **구독 만료 처리 로직** — 30일 무료 체험 종료 후 기능 제한이 실제로 동작하는지 E2E 테스트. `usageLimitService.resetMonthlyReservationCounts()`에 TODO 주석이 남아 있음.

3. **데이터 백업/복구 계획** — 네이버 클라우드 PostgreSQL 자동 백업 설정 확인.

4. **HTTPS/SSL 인증서** — 프로덕션 배포 시 SSL 필수 (카카오 OAuth, PG 연동 모두 HTTPS 요구).

5. **이미지 업로드 경로** — 현재 `C:/Project/uploads/` 로컬 저장. 프로덕션에서는 네이버 클라우드 Object Storage로 전환 필요.

6. **캘린더 영업시간** — 10:00-20:00 하드코딩 잔재가 남아있을 수 있음. 매장설정 값 연동 재확인.

7. **월간 예약 카운트 초기화 배치** — `resetMonthlyReservationCounts()` TODO → 스케줄러 구현 필요.

### 4.2 💡 B2B2C SaaS 관점 핵심 조언

1. **첫 매장 성공 경험이 최우선** — 10곳 베타 테스터 확보가 목표라면, 1곳이 완벽하게 사용할 수 있는 게 10곳이 반쯤 쓰는 것보다 나음. 미용실 1곳에서 실사용 테스트 후 피드백 반영.

2. **고객(C) 경험 = 매장(B) 만족** — 매장 관리자가 만족하려면 그들의 고객이 편하게 예약해야 함. 슬러그 페이지 접속 → 예약 완료까지의 UX가 핵심.

3. **알림은 킬러 기능** — 미용실 입장에서 예약 노쇼가 가장 큰 고통. 카카오 알림톡이 늦어지더라도, 최소 이메일이라도 예약 확인 알림을 보내는 것이 좋음.

4. **가격 정책 재검토** — 무료 플랜에 "광고 표시"보다는 기능 제한이 더 효과적. 소상공인은 광고가 보이는 서비스를 고객에게 보여주기 싫어함.

> 균모 님, 1인 개발 3개월 만에 이 정도 완성도면 정말 잘 하고 있습니다. 핵심 기능(예약/고객/스태프/서비스/통계/리뷰)이 모두 돌아가는 상태이고, 고객 예약 플로우(B2C)까지 완성된 건 큰 진전입니다.
